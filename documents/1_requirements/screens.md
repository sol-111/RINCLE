# Rincle 機能一覧・詳細

Bubble アプリケーション `rincle.bubble` に定義されている全画面と各画面の機能一覧、および改善点です。

## サマリー

| 項目 | 値 |
|------|-----|
| 総画面数 | 14 |
| 総 UI 要素数 | 136（Bubbleファイル定義値） |
| 総ワークフロー数 | 530 |

## 全画面共通の改善点

### UI/UX 観点

| # | 改善点 | 優先度 |
|---|--------|--------|
| 1 | ページタイトル・meta description・OGP タグが未設定のため、SNS シェア時にリンクプレビューが表示されず、SEO にも悪影響 | 高 |
| 2 | viewport meta タグの設定が確認できず、スマートフォンでの表示が崩れる可能性がある | 高 |
| 3 | 画像に alt テキストが設定されていないため、スクリーンリーダー利用者がコンテンツを認識できない | 中 |
| 4 | ARIA ラベルが未設定でキーボードナビゲーションが機能しない。WCAG 準拠が困難な状態 | 中 |
| 5 | CDN リソースの読み込み失敗時に「IT 管理者に連絡してください」という不親切なエラーメッセージが表示される | 中 |
| 6 | 広告ブロッカー利用環境でアプリが起動しないケースがある | 中 |
| 7 | エラー通知をポップアップに集中させており（63 個）、操作の流れを頻繁に中断する。トースト通知・インライン表示への切り替えを検討 | 低 |
| 8 | 空のリスト表示時のエンプティステート（「該当なし」等のメッセージ）が未定義 | 低 |

### 開発・セキュリティ観点

| # | 改善点 | 優先度 |
|---|--------|--------|
| 1 | **【緊急】** Pay.JP の本番公開鍵・テスト秘密鍵がオプションセットにハードコードされており、JSON 書き出し等で漏洩するリスクがある。即時ローテーションが必要 | 緊急 |
| 2 | 16 本の API ワークフローで `ignore_privacy_rules: TRUE` が設定されており、決済・予約データへのアクセス制御が無効化されている | 緊急 |
| 3 | 複数の決済関連 API で `auth_unnecessary: TRUE` が設定されており、未認証リクエストが受け入れられる | 緊急 |
| 4 | `birth_date`・`phone_text`・`address_text`・`car_id_text`・`cus_id_text` 等、個人情報 11 フィールドにプライバシーロールが未設定 | 高 |
| 5 | Pay.JP Webhook の署名検証が確認できず、偽造リクエストを受け入れる可能性がある | 高 |
| 6 | Google Maps API キーがフロントエンドに露出している。ドメイン制限等のリファラー制限を設定する必要がある | 高 |
| 7 | メールアドレス 5 件・URL 9 件がワークフロー内にハードコードされており、環境切り替えや変更時の修正漏れリスクがある | 中 |
| 8 | データタイプ名・要素 ID に `___`、`bTHEn` 等の無意味な名前が多数使われており、保守性が著しく低い | 中 |
| 9 | プロジェクト全体でコメントが 1 件のみ。ワークフローの意図・ロジックが不明瞭で引き継ぎ困難 | 中 |
| 10 | 全銀ネット（zengin.ajtw.net）との連携があるが、銀行口座情報の取り扱いにおける PCI DSS 準拠状況が不明 | 中 |
| 11 | クライアントサイドの入力バリデーションが確認できず、不正な値がそのまま送信される可能性がある | 中 |
