## 1. 店舗在庫作成・更新ケース

店舗の営業カレンダーは **常に1年分（365日以上）** 自動で管理される。
毎日バッチで1日分が追加され、店舗が曜日設定を行うと作成日から1年分の日付データが生成される。

### システム（バッチ処理）

| # | テストケース |
|---|---|
| 1 | 本日から365日以上分のデフォルト日程が存在していること |
| 2 | デフォルトが毎日1日分ずつ追加されること（翌日になると1日分追加される） |
| 3 | 店舗の日付も毎日1日分ずつ追加されること |

### 店舗（初期設定・変更時）

| # | テストケース |
|---|---|
| 1 | 店舗の曜日を初期設定したとき、デフォルト分の日付データが作成されること |
| 2 | 曜日設定を変更したとき、翌日以降の店舗日程が新しい設定に更新されること |
| 3 | 特定の日付を個別に変更したとき、その日付のデータが更新されること |

### 自転車（カレンダー連動）

| # | テストケース |
|---|---|
| 1 | 店舗カレンダーが更新されたとき、自転車の貸し出し可能日も仕様通りに更新されること |
| 2 | 自転車を新規作成したとき、デフォルト分の貸し出し可能日が自動作成されること |

## 2. キャンセルケース

ユーザーと店舗では **キャンセルできるタイミングが異なる**。
ユーザーはライド前日の23:59:59まで、店舗はライド当日でもキャンセル可能。
返金の有無は常に店舗側が選択できる。

### ユーザー側

| # | テストケース |
|---|---|
| 1 | ライド前日の23:59:59まではキャンセルボタンが操作できること |
| 2 | ライド当日（0:00以降）はキャンセルボタンが表示されるが非活性になること |

### 店舗側

| # | テストケース |
|---|---|
| 1 | 予約が「来客待ち」ステータスのときにキャンセルが実行できること |
| 2 | 返金有無の選択が常に入力できる状態であること |

## 3. 決済ケース

決済方法はクレカ・現地決済の2種類。クレカはPay.jpを経由する。
決済が完了・キャンセル・延長・短縮されたとき、在庫にも正しく反映される必要がある。

### 決済方法（全パターン確認）

| # | テストケース |
|---|---|
| 1 | クレカ決済が成功するパターン |
| 2 | クレカ決済が失敗するパターン |
| 3 | クレカで初回決済後、延長時に現地決済に切り替えるパターン |
| 4 | 現地決済のみで完結するパターン（初回のみ） |
| 5 | 現地決済で初回決済後、延長も現地決済するパターン |

### 在庫反映（各操作後の確認）

| # | タイミング | 確認内容 |
|---|---|---|
| 1 | 店舗がキャンセルしたとき | 対象日程の在庫が空きに戻ること |
| 2 | ユーザーがキャンセルしたとき | 対象日程の在庫が空きに戻ること |
| 3 | 延長したとき | 延長分の日程が在庫に反映されること |
| 4 | 期間を短縮したとき | 短縮分の日程が在庫に戻ること |

## 4. 自転車料金計算

貸出時間と返却時間の差によって、適用されるプランと料金が自動的に決まる。
自転車詳細画面の条件Dropdownには営業時間のみが表示される。

### 4-1. 日付指定：可の場合（プラン判定ロジック）

返却日時と貸出日時の差分から、以下のプランが自動適用される。

| プラン | 料金（税抜） | 適用条件 |
|---|---|---|
| 〜3時間 | ¥880 | 1時間 ≦ 返却日時 - 貸出日時 ＜ 4時間 |
| 4時間以上〜1日 | ¥3,300 | 4時間 ≦ 返却日時 - 貸出日時 ≦ 貸出日の営業終了時間 |
| 1泊（2日間） | ¥13,200 | 返却日の日付 - 貸出日の日付 = 1日 |
| 2泊（3日間） | ¥27,500 | 返却日の日付 - 貸出日の日付 = 2日 |
| 3泊〜6泊（〜7日間） | ¥27,500 | 返却日の日付 - 貸出日の日付 ≧ 3日 |
| 7泊〜13泊（〜14日間） | ¥27,500 | 返却日の日付 - 貸出日の日付 ≧ 7日 |
| 14泊〜29泊（〜30日間） | ¥27,500 | 返却日の日付 - 貸出日の日付 ≧ 14日 |

### 4-2. 日付指定：不可の場合

以下の条件に1つでも該当する場合、その日程は予約不可として扱われる。

| # | 条件 |
|---|---|
| 1 | 貸出日または返却日が店舗の休日である |
| 2 | 貸出日〜返却日の期間内に他のレンタル予定が含まれている |
| 3 | 貸出日を起算して31日以上のレンタルである |
| 4 | 店舗が「バイクレンタル不可」に設定している自転車である |

### 4-3. 貸し出し可能期間の設定

| # | テストケース |
|---|---|
| 1 | 期間指定不可の設定になっている場合、予約できないこと |
| 2 | レンタル可能期間に応じて、表示される料金が切り替わること |

## 5. 延長料金計算

延長は **返却日が同日かどうか・営業終了時間内かどうか** で計算単位が変わる。
延長料金は返却時にまとめて「延長料金」として別途決済される。

### 5-1. 延長時の計算単位

| 返却タイミング | 計算単位 | テストケース |
|---|---|---|
| 返却日が同日かつ営業終了時間前 | 1時間単位 | 同日の営業時間内に返却する延長 |
| 返却日が同日かつ営業終了時間以降 | 泊数 × 日料金 | 同日だが営業終了時間を超える延長 |

> ＊期間内に別の予約が入っている場合は、前日までしか延長できない

### 5-2. 決済方法

| # | テストケース |
|---|---|
| 1 | 延長の決済は現地決済のみで処理されること |

### 5-3. 延長不可となる条件（4-2と同条件）

以下の条件に該当する場合、延長が不可になることを確認する。

| # | 条件 |
|---|---|
| 1 | 貸出日または返却日が店舗の休日である |
| 2 | 延長後の期間内に他のレンタル予定が含まれている |
| 3 | 貸出日起算で31日以上になるレンタル |
| 4 | 店舗が「バイクレンタル不可」に設定している自転車である |

## 6. 料金の変更

料金テーブルを変更しても、**すでに作成済みの予約には影響しない**。
変更後に新規作成された予約にのみ、新しい料金が適用される。

| # | テストケース |
|---|---|
| 1 | 料金改定前に作成された予約には、改定前の料金テーブルが適用されること |
| 2 | 料金改定後に新規作成した予約には、改定後の料金テーブルが適用されること |
| 3 | 貸し出し中の自転車を延長するとき、改定前の料金テーブルが使われること |

## 7. 検索のケース

貸出日・返却日の指定有無によって、検索結果に表示される自転車が変わる。
「貸出不可」設定の自転車は、どの条件でも常に非表示。

### 検索パターン別の表示ルール

| 貸出日 | 返却日 | 全て空きあり | 指定日空きあり | 空き無し | 貸出不可 |
|:---:|:---:|:---:|:---:|:---:|:---:|
| 指定 | 指定 | ○ | × | × | × |
| 未定 | 指定 | ○ | ○ | × | × |
| 指定 | 未定 | ○ | ○ | × | × |
| 未定 | 未定 | ○ | ○ | ○ | × |

> 両日指定：完全に空きがある自転車のみ表示
> 片方のみ指定：指定した日に空きがあれば表示
> 両日未定：空きが1日でもあれば表示（空き無しの非活性状態も含む）

### 検索状態の詳細ロジック

**自転車の表示可否は「貸出可否」×「日程かぶり」で決まる**

| | パターン1 | パターン2 | パターン3 | パターン4 |

| **日程かぶり判定** | | | | |
| あり（他の予約が被っている） | ● | | ● | |
| なし（他の予約が被っていない） | | ● | | ● |
| **表示結果** | | | | |
| 空きあり（通常表示） | | ● | | |

## 8. カート機能の削除

カートは **当日限り有効**。翌日になるか、ログインしていない状態でページを開くと自動的に削除される。
削除時はアラートを表示した後、空車検索画面へ遷移する（入力途中の情報は都道府県のみ引き継ぐ）。

### 削除トリガー

| # | 条件 | 動作 |
|---|---|---|
| 1 | カート作成日の翌日にページをロードしたとき（例：1/1作成 → 1/2アクセス） | カートを削除 |
| 2 | ログインしていないユーザーがページをロードしたとき | アラートを表示してカートを削除 |

**対象画面**：自転車詳細画面・カート画面（カートに1台以上の自転車がある場合のみアラート表示）

### カート削除後の遷移先

空車検索画面へ遷移し、以下の状態で表示する。

| 項目 | 値 | 備考 |
|---|---|---|
| 都道府県 | カート作成時の値 | 引き継ぐ |
| カテゴリ | 全て | リセット |
| 貸出日 | 未定 | リセット |
| 返却日 | 未定 | リセット |
