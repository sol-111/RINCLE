## 第2回（2026年2月3日）

**参加者:** 松永光希、森藤哲彰

### 話し合った内容

| # | テーマ | 内容 |
|---|--------|------|
| 1 | プロダクト品質 | 視察の結果、想定より悪くない状態と評価。ただし要件定義・仕様把握にコストがかかると判明 |

| 5 | Bubbleの活用方針 | Bubbleのフロントと外部DB（Supabase等）を組み合わせる構成がパフォーマンス向上に有効と紹介 |
| 6 | 森藤氏の会社体制 | 正社員10名・業務委託30名の計40名体制。2025年11月に上場企業（アイビス社）傘下に統合済み |

| 内容 |
|------|
| ゼロから作り替えはせず、現行システムの改善ルートで進める |
| 準委任契約（時給契約）での進行で合意 |
| 初期アサイン：PM + エンジニア2名 |
| 問題点の言語化と優先度付けから着手 |

### 主要課題

| カテゴリ | 課題 |
|----------|------|
| 開発品質 | 要件定義なしのアジャイル開発によるバグ多発・テスト不足 |
| パフォーマンス | 検索・トップページの表示速度の遅延（5秒以上） |
| コード品質 | 金額計算ロジック等が人間に読めない複雑な実装状態 |
| ベンダー管理 | 初期ベンダーの体制変更・責任所在の曖昧さ・連絡不通リスク |
| スケーラビリティ | 100〜300店舗対応に向けたDBおよびアーキテクチャ設計の必要性 |
| ビジネス | 競合出現への危機感・資金調達の判断・ベンダー選定の遅れ |

### キーワード

- **改修方針:** ゼロから作り替えず、現行Bubbleシステムを改善
- **契約形態:** 準委任契約（時給）＋成功報酬型も検討
- **技術方針:** Bubbleフロント ＋ 外部DB（Supabase等）の組み合わせが有力
- **体制:** PM・テックリードの2名体制からスタートし、優先度確定後に追加投入
