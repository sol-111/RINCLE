## 補足メモ（優先度・用語）

### 1. 開発環境のパスワードなし（#7）

表では **緊急** とした。テスト版 URL が無防備で、かつ本番相当のデータ・キーが載る場合、URL を知られた第三者が画面や挙動を観察できるリスクが高いため。

### 2. 「HTTP 200」と `admin_update_calendar`、§2-4 との関係（#4）

- **HTTP 200** は、その URL が「正常にページとして返った」こと。未ログインで管理系が **200** だと、Bubble のセキュリティチェックではセンシティブページ不備になりやすい。
- `**page_is_private`** … ログインユーザーのみがページを開ける設定。オフだと未ログインでもページ配信になり得る。
- **WF リダイレクト** … Page is loaded で未ログインなら別ページへ。**一瞬ページが開く**ため、private と併用が推奨されやすい。
- **#4** に、コード側（S-105・API 認可）と Bubble 側（センシティブ各ページのリダイレクト）を **1 行で統合**した。§2-4 はこのチケットの中心。

### 3. API Connector の URL ハードコード（#20）

Bubble Issue どおり Private パラメータ化。**優先度は中**。#17,#35 は別チケット。

### 4. API エラー未捕捉（#17）

**高**。課金・キャンセル・返金で失敗が見えないと金銭・予約不整合につながるため。

### 5. `admin_login`（#4 と #37）

#4 でページ・リダイレクト・API 認可を扱い、#37 が WF・権限・WU（不要検索）の詳細。**ワンセットで確認**すると漏れが減る。
