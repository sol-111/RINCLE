PAY.JP API リファレンス

PAY.JP REST API (v1) の全リソース・エンドポイントを日本語で整理したリファレンスです。移行・設計のたたき台としてご利用ください。

Base URL

https://api.pay.jp/v1/

Protocol

HTTPS のみ

Authentication

Basic認証 (sk_...)

Response

JSON (UTF-8)

1

全体像

項目内容
ベース URLhttps://api.pay.jp/v1/
プロトコルHTTPS のみ
認証Basic 認証 — ユーザー名にシークレットキー（sk_...）、パスワードは空
パブリックキーpk_... — 主にブラウザ側でカードをトークン化するときに使用
メソッドGET POST DELETE
POST のボディContent-Type: application/x-www-form-urlencoded
レスポンスJSON（UTF-8）
📖
API v2 について: PAY.JP v2 ドキュメントでは、クレジット以外の手段や Checkout 等が扱われます。本ドキュメントは v1 REST のリソース中心です。

共通機能（リソース以外）

区分内容
Errorエラーは JSON の error オブジェクト（type, code, message, param 等）。SDK では用途別例外で捕捉可能
Rate Limitレートリミットゾーンあり。超過時は 429 等
Listlimit / offset 等で一覧 API をページ分割
Metadata各オブジェクトに付与可能なメタデータ（キー数・文字数に上限あり）
Event / Webhookリソース操作時に event が生成され、登録先があれば Webhook 送信

Platform API BETA

マーケットプレイス型・Payouts 型アカウント向け。テナントごとに売上分配・入金・プラットフォーム手数料などを扱います。

Platform API の特徴

  • 通常の Charge / Transfer を拡張して利用する部分と、Tenant / TenantTransfer 専用リソースがある
  • Platform 利用時、Plan / Subscription の一部 API は利用不可（公式に「準備中」と記載）という制約あり
  • 認証・JSON 形式は v1 共通項目に準拠。Platform 固有の error[code] が追加される
🏢

Charge / Transfer for Platform

Platform
リソース説明

Transfer プラットフォーマー側の入金は通常の /v1/transfers。テナント側の入金は /v1/tenant_transfers

API Connector にはテナント / 顧客 / カード / トークン / 課金 / 返金など api.pay.jp/v1/... の呼び出しがまとまって定義されています。
