# 99_receives 仕様 未確認事項

受領資料（scope.html / test_cases.html / フロー図HTML / PDF業務フロー / payjp_integration.md）を横断的にレビューした結果、以下の矛盾・曖昧点を確認。実装着手前にクライアント or 浅井さんへの確認が必要。

---

## 1. 延長決済の方法が矛盾している【優先度: 高】

| 資料 | 記載内容 |
|------|----------|
| scope.html | 延長料金は返却時にまとめて**現地決済** |
| test_cases.html | 延長の決済は**現地決済のみ**で処理される |
| payjp_integration.md (セクション4) | 延長料金もPay.JP経由でカード引き落としするフローが記載 |
| image.png（業務フロー図） | 延長後の返却時に「現地決済」ルートのみ |

**確認事項:** 延長はクレカ決済もあり得るのか、現地決済のみなのか？

---

## 2. 決済がオーソリ→キャプチャなのか直接決済なのか【優先度: 高】

| 資料 | 記載内容 |
|------|----------|
| flow_credit.html | 予約時にオーソリ（与信枠確保・有効期限30日）→ ライド開始時にキャプチャ（決済確定）の2ステップ |
| payjp_integration.md | API一覧に `Charge`（直接決済）のみ記載、auth/captureの分離なし |
| scope.html | 「本決済が始動する」とだけ記載、オーソリ/キャプチャの区別なし |

**確認事項:** 予約時にオーソリを取るのか、それとも直接決済なのか？ Pay.JPのプラットフォーム決済でauth/captureがサポートされているか要確認。

---

## 3. 料金テーブルの金額は固定値か店舗ごとか【優先度: 中】

| 資料 | 記載内容 |
|------|----------|
| test_cases.html | 具体的な金額（~3時間=¥880, 4時間~1日=¥3,300, 1泊=¥13,200 等） |
| flow_bike_reg.html | 店舗がバイク登録時に「適用する料金表選択」→ デフォルト値を元に修正可能 |
| scope.html | 管理画面でプランごとの金額を編集可能 |

**確認事項:** テストケースの金額はデフォルト値の例であり、実際は店舗・バイクごとに異なる認識でよいか？

---

## 4. 2泊以降の料金がすべて ¥27,500 で同一【優先度: 中】

test_cases.html の料金表:

| プラン | 料金（税抜） |
|--------|-------------|
| 1泊（2日間） | ¥13,200 |
| 2泊（3日間） | ¥27,500 |
| 3泊〜6泊 | ¥27,500 |
| 7泊〜13泊 | ¥27,500 |
| 14泊〜29泊 | ¥27,500 |

**確認事項:** 意図的に全期間同額なのか、テストデータの仮置きなのか？ 同額ならプラン分けの意味がない。

## 5. ユーザー発キャンセル時の返金は自動か店舗判断か【優先度: 高】

| 資料 | 記載内容 |
|------|----------|
| scope.html / test_cases.html | ユーザーは前日 23:59:59 までキャンセル可 |
| image-2.png | 店舗側キャンセルUIで「レンタル料金を請求しない / する」を選択 |
| payjp_integration.md | 返金は管理画面からの**手動操作**（お客さん側からは返金できない） |

**確認事項:** ユーザーが自分でキャンセルした場合、返金は自動で行われるのか？ それとも店舗 or 管理者が手動で返金処理するのか？

## 6. 延長の起点はユーザー（オンライン）か店舗（電話）か【優先度: 中】

| 資料 | 記載内容 |
|------|----------|
| scope.html UC2 | 「マイページで延長を選択し、追加時間を入力する」→ ユーザー起点 |
| flow_bike_rental.html | 「延長したい → 電話で店舗に連絡及び確認」→ 店舗起点（電話ベース） |

**確認事項:** 両方のルートがあるのか？ オンライン延長と電話延長では、在庫チェック・決済のフローが異なるはず。

## 7. 「期間短縮」のフローが未定義【優先度: 低】

scope.html 在庫反映ルール: 「期間を短縮した場合 → 返却済みステータスになった予定のみ、在庫の変更が可能」

**確認事項:** 期間短縮 = 予定より早く返却しただけか？ それとも明示的に「期間変更」UIがあるか？ 差額返金はどうなるか？
